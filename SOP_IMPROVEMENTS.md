# LearnAI SOP 與功能優化建議

本文件彙整多方利害關係人模型 review 結論，作為下一輪學生與老師功能優先序。

## 本輪已落地

- Chat 課程範圍：選擇課程後，文件清單改為該課程 ready 教材；後端也禁止 course session 混入私人文件。
- 文件上傳：支援一次選取多檔，上傳 API 逐檔回報成功或失敗。
- 老師課程管理：可編輯課程、重置邀請碼、調整成員角色、移除非 owner 成員、管理課程教材。
- Admin 全域管理：補強文件、對話、課程管理；敏感讀取寫入 audit；禁止移除課程 owner；課程教材共享需 ready 且文件 owner 是課程成員。
- 學生錯題本：改為依最新一次作答只列錯題，附上學生答案與正解。

## 學生 SOP

1. 上傳多份教材，等待狀態成為 ready。
2. 針對單份教材查看摘要、心智圖與覆蓋度。
3. 在 Chat 選擇個人文件或課程範圍，先用課程全部教材問答，再縮小到特定文件。
4. 生成測驗並作答，錯題自動進入錯題本。
5. 針對錯題與薄弱章節回到 Chat 追問，整理成筆記。

## 老師 SOP

1. 建立課程並邀請學生加入。
2. 上傳教材並加入課程；課程教材只使用 ready 文件。
3. 管理成員角色，助教或共同教師設為 instructor。
4. 查看課程進度，先以課程 chat sessions 和課程筆記作為活動指標。
5. 針對低活動學生或高頻問題，補充教材、建立測驗或調整課程節奏。

## 下一輪 P1

- 課程作業與教師測驗：老師可指定文件、截止日、題數與難度，學生提交後老師看彙總。
- 課程公告與任務清單：把「本週讀哪些文件、完成哪些測驗」明確放進課程首頁。
- 課程內筆記/閃卡歸屬：新增 `course_id`，讓老師進度只統計課程活動，不碰學生私人學習資料。
- 弱點儀表板：依錯題、引用頁面、chat 主題聚合出學生薄弱概念。

## 下一輪 P2

- 文件批次處理佇列頁：顯示每個檔案 OCR/embedding 進度、失敗原因、重試按鈕。
- 學生學習路徑：依文件章節、錯題與目標日期產生每日任務。
- 教材版本管理：老師更新文件時保留舊版引用與課程公告。
- Admin break-glass：讀取私人對話或文件 detail 時要求理由欄位，並在 audit 中保留。
